1. COLLOCAZIONE E FLUSSO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Il livello bonus appare una volta per playthrough, dopo L5, per tutte
le difficoltà. Non è in LEVELS[]: l'array rimane 10 elementi, la
win condition `currentLevel === LEVELS.length` è invariata.

Flusso:
  L5 completato → nextLevel() → bonusActive = true
                                currentLevel rimane 5
                                carica BONUS_LEVEL
  Timer bonus scade → bonusActive = false
                      currentLevel = 6
                      startLevel() normale

4. LUCA — SPRITE GIOCABILE (draw-chars.js)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

COLOURS_LUCA e drawLucaEnd() esistono già in draw-chars.js.
I colori PAL.lucaXxx esistono già in palette.js. Nessuna nuova
voce di palette necessaria per Luca.

- In bonusActive, il game loop usa drawChar(..., COLOURS_LUCA)
  invece di drawChar(..., COLOURS_MARCO).
- Animazione lancio: braccio alzato per ~18 frame (0.3s a 60fps)
  prima che il proiettile venga spawnato. Flag: player.throwAnim > 0.
- Luca NON perde vite se catturato dai professori durante il bonus.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
5. PROIETTILE FISICO (physics.js + entities.js)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

spawn: quando player preme azione + bonusActive + player.throwAnim === 0
  → player.throwAnim = 18
  → dopo 18 frame: push in paperProjectiles[]
    { x: player.x + (dir*4), y: player.y, dir: player.dir, decay: 24 }

update (ogni frame, in updateBonus()):
  per ogni proiettile:
    p.x += p.dir * 2  (velocità 2px/frame)
    p.decay--
    if (p.decay <= 0) → rimuovi
    collision con teacher: teacher.knockedT = 180 (3s), rimuovi proiettile
    collision con wanderer: wanderer → stato 'tripped', rimuovi proiettile

Decay = 24px = 3 "tile" da 8px.

6. WANDERING AI (entities.js)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Ogni wanderer ha: { x, y, floor, dir, wanderTimer, state, knockedT,
                    rollDir, rollT, chainHit, shirtColor }

shirtColor: assegnato alla creazione dal pool PAL.wandererShirt[1..7].
palette.js — nuove voci in sezione GAME OBJECTS o CHARACTERS:
  wandererShirt1 .. wandererShirt7  (7 colori distinti per le maglie)

stati: 'walking' | 'tripped' | 'rolling'

walking:
  x += dir * 0.5  (più lento dei prof)
  wanderTimer--
  if (wanderTimer <= 0): dir *= -1, wanderTimer = rand(120, 180)
  rimbalza sui bordi del floor corrente (usa minX/maxX per piano)

tripped (studente inciampa su prof a terra):
  knockedT = 60 (1s di animazione caduta)
  controllare se entro 30px da un buco scala → stato 'rolling'
  altrimenti → assegnare punti, tornare a 'walking' dopo knockedT

rolling (rotola giù per le scale):
  rollT = 90 (1.5s)
  scende visivamente lungo la scala più vicina
  al termine: torna a 'walking' sul piano inferiore
  premio: +1 vita, floating text "+1 VITA!"

TRIGGER tripped: ogni frame, se teacher.knockedT > 0, controllare
  se un wanderer 'walking' si sovrappone all'hitbox del prof
  (|wanderer.x - teacher.x| < 8 && stesso piano).

7. CARAMBOLA E CATENA (entities.js)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Ogni evento 'tripped' su wanderer lontano dalle scale:
  bonusCarambole++
  bonusBonusScore += 500 * chainMultiplier
  floating text "CARAMBOLA!" + punteggio
  chainMultiplier = 2 per i successivi eventi entro 90 frame

chainMultiplier va in state.js: var bonusChainMult = 1
reset: bonusChainMult = 1 quando nessun wanderer è in stato
  tripped/rolling da più di 90 frame.

Ogni wanderer ha flag chainHit = false; quando diventa tripped:
  if (un altro wanderer nelle vicinanze è già tripped/rolling):
    chainHit = true → il moltiplicatore rimane 2 per questo evento

Rolling (caduta scala): non usa chainMultiplier, premia sempre +1 vita.

12. EDGE CASES DA GESTIRE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Se Luca viene catturato da un prof durante il bonus: NON perdere vita,
  ma il prof lo rallenta (knockback leggero) — o ignorare completamente
  la cattura (professor non insegue Luca nel bonus?).
  DECISIONE APERTA: i prof pattugliano normalmente o no?

- highscores: il bonus non è un "livello" a sé, i punti
  si sommano allo score corrente prima di salvare in _addHighScore.

- restartGame() deve resettare bonusActive=false così che al restart
  il bonus appaia di nuovo dopo L5.

- Se il giocatore completa il bonus senza fare nessuna carambola:
  banner mostra "CARAMBOLE: 0", nessuna vita/punti bonus.
  Il gioco prosegue normalmente.
