## Note importanti

- layout.js viene ELIMINATO — non rinominato. gfx-layout.js è un file nuovo.
  Questo perché git rename + modifica contestuale in una commit sono più puliti
  rispetto a modify-in-place con nomi temporanei.

- C = CONFIG.colors in state.js NON cambia. gfx-palette.js carica prima di state.js
  quindi CONFIG.colors è disponibile quando state.js esegue const C = CONFIG.colors.

- PAL è definita in gfx-palette.js ma non referenziata da nessun altro file in Fase 1.
  Serve solo come riferimento pronto per Fase 2.

- draw.js rimane con il nome attuale (non diventa gfx-draw.js) per minimizzare le
  modifiche al resto del codice che dipende da draw.js.

- Alla fine di Fase 1 il gioco deve essere identico a prima: stessi colori, stesso
  comportamento, stessa performance.
