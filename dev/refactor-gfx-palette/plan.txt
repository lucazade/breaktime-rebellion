## Nota tecnica: PW/PH e playerStart

PW/PH si definiscono in gfx-chars.js (dopo gfx-layout.js).
playerStart si aggiunge a CONFIG.vis.shared in gfx-chars.js dopo aver definito PH:

  // gfx-layout.js — shared senza playerStart
  shared: { stairs: [...], boards: [...], desks: [...], bell: {...} }

  // gfx-chars.js — PH disponibile qui, aggiungiamo playerStart all'oggetto esistente
  const PW = 8, PH = 16;
  CONFIG.vis.shared.playerStart = { x: 18, y: GY - PH - walkOffset };

SHARED_LAYOUT è un riferimento all'oggetto — levels.js e state.js (che caricano dopo) vedono
playerStart perché è lo stesso oggetto in memoria.
